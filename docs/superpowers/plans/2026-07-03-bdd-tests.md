# BDD Tests Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the Reqnroll + xUnit BDD test project that exercises the full business scenarios of the MCP Gateway end-to-end through Gherkin feature files: tool generation, tool call proxy, spec refresh, auth strategies, and client profile schema transformations.

**Architecture:** A dedicated `McpGateway.BddTests` xUnit project that hosts Reqnroll feature files in `Features/` and step definitions in `Steps/`. A single `TestRunHooks` class wires up shared infrastructure: an `HttpClient` pointed at the gateway's `WebApplicationFactory`, a Testcontainers PostgreSQL instance for persistence, a Testcontainers Azurite instance for audit emission, and WireMock.Net servers that mock the underlying APIs and Entra ID's OBO token endpoint. Reqnroll discovers the `[Binding]` classes and generates xUnit test cases per scenario. Each scenario drives a real HTTP request through the gateway and asserts on the response, the database state, and the WireMock call log.

**Tech Stack:** .NET 10, Reqnroll 2.4.0 + Reqnroll.xUnit 2.4.0, xUnit 2.9.0, Microsoft.AspNetCore.Mvc.Testing 10.0.0, Testcontainers.PostgreSql 4.13.0, Testcontainers.Azurite 4.13.0, WireMock.Net 1.6.7, FluentAssertions 8.2.0, System.IdentityModel.Tokens.Jwt 8.19.1.

Create `tests/McpGateway.BddTests/Support/JwtTestSigner.cs`:

```csharp
using System.IdentityModel.Tokens.Jwt;
using System.Security.Claims;
using System.Security.Cryptography;
using Microsoft.IdentityModel.Tokens;

namespace McpGateway.BddTests.Support;

public sealed class JwtTestSigner
{
    public string KeyId { get; }
    public string Issuer { get; }
    public string Audience { get; }
    public RSA PrivateKey { get; }
    public string JwksJson { get; }

    public JwtTestSigner(string keyId, string issuer, string audience)
    {
        KeyId = keyId;
        Issuer = issuer;
        Audience = audience;
        PrivateKey = RSA.Create(2048);
        JwksJson = BuildJwks();
    }

    public string SignToken(string subject, IEnumerable<Claim>? extraClaims = null)
    {
        var key = new RsaSecurityKey(PrivateKey) { KeyId = KeyId };
        var creds = new SigningCredentials(key, SecurityAlgorithms.RsaSha256);
        var claims = new List<Claim>
        {
            new(JwtRegisteredClaimNames.Sub, subject),
            new(JwtRegisteredClaimNames.Oid, subject),
            new("preferred_username", $"{subject}@example.com")
        };
        if (extraClaims is not null)
        {
            claims.AddRange(extraClaims);
        }

        var token = new JwtSecurityToken(
            issuer: Issuer,
            audience: Audience,
            claims: claims,
            notBefore: DateTime.UtcNow,
            expires: DateTime.UtcNow.AddHours(1),
            signingCredentials: creds);

        return new JwtSecurityTokenHandler().WriteToken(token);
    }

    private string BuildJwks()
    {
        var parameters = PrivateKey.ExportParameters(includePrivateParameters: false);
        var jwk = new
        {
            kty = "RSA",
            use = "sig",
            kid = KeyId,
            alg = "RS256",
            n = Base64UrlEncoder.Encode(parameters.Modulus!),
            e = Base64UrlEncoder.Encode(parameters.Exponent!)
        };
        return "{\"keys\":[" + System.Text.Json.JsonSerializer.Serialize(jwk) + "]}";
    }
}
```

Create `tests/McpGateway.BddTests/Support/EntraIdMock.cs`:

```csharp
using WireMock.RequestBuilders;
using WireMock.ResponseBuilders;
using WireMock.Server;

namespace McpGateway.BddTests.Support;

public sealed class EntraIdMock : IAsyncDisposable
{
    public WireMockServer Server { get; }
    public string Authority { get; }
    public string TokenEndpoint { get; }
    public string TenantId { get; }
    public string ClientId { get; }

    public EntraIdMock(string tenantId, string clientId)
    {
        Server = WireMockServer.Start();
        Authority = Server.Url!;
        TokenEndpoint = $"{Authority}/{tenantId}/oauth2/v2.0/token";
        TenantId = tenantId;
        ClientId = clientId;

        Server
            .Given(Request.Create().WithPath($"/{tenantId}/oauth2/v2.0/token").UsingPost())
            .RespondWith(Response.Create()
                .WithStatusCode(200)
                .WithHeader("Content-Type", "application/json")
                .WithBodyAsJson(new
                {
                    access_token = "obo-m2m-token",
                    expires_in = 3600,
                    token_type = "Bearer"
                }));
    }

    public string Value => $"https://{new Uri(Authority).Host}/{TenantId}/v2.0";

    public async ValueTask DisposeAsync()
    {
        Server.Stop();
        await Server.DisposeAsync();
    }
}
```

Note: `Value` is used to configure `Jwt:Authority` in the test host so that the gateway's JWT handler trusts the mock's signing key. The actual issuer claim is the WireMock URL (e.g., `http://localhost:1234/{tenantId}/v2.0`) because `JwtAuthHandler` in the auth plan sets the expected issuer to the configured authority.

```csharp
using McpGateway.BddTests.Steps;
using McpGateway.BddTests.Support;
using Reqnroll;
using Testcontainers.Azurite;
using Testcontainers.PostgreSql;
using WireMock.Server;

namespace McpGateway.BddTests.Hooks;

[Binding]
public sealed class TestRunHooks
{
    private static readonly PostgreSqlContainer Postgres = new PostgreSqlBuilder()
        .WithImage("postgres:18-alpine")
        .WithDatabase("mcp_gateway_bdd")
        .WithUsername("mcp")
        .WithPassword("mcp")
        .Build();

    private static readonly AzuriteContainer Azurite = new AzuriteBuilder()
        .WithImage("mcr.microsoft.com/azure-storage/azurite:3.34.0")
        .Build();

    private static bool _infrastructureStarted;
    private static readonly SemaphoreSlim InitLock = new(1, 1);

    private readonly TestContext _context;

    public TestRunHooks(TestContext context)
    {
        _context = context;
    }

    [BeforeTestRun]
    public static async Task BeforeTestRun()
    {
        await InitLock.WaitAsync();
        try
        {
            if (_infrastructureStarted)
            {
                return;
            }
            await Postgres.StartAsync();
            await Azurite.StartAsync();
            _infrastructureStarted = true;
        }
        finally
        {
            InitLock.Release();
        }
    }

    [AfterTestRun]
    public static async Task AfterTestRun()
    {
        await Postgres.DisposeAsync();
        await Azurite.DisposeAsync();
    }

    [BeforeScenario]
    public async Task BeforeScenario(ScenarioContext scenarioContext)
    {
        var signer = new JwtTestSigner(
            keyId: "test-key-1",
            issuer: $"https://{new Uri(Postgres.ConnectionString).Host}/test-tenant/v2.0",
```

```csharp
using McpGateway.Api.Auth;
using McpGateway.Api.Endpoints;
using McpGateway.Core;
using McpGateway.Core.Audit;
using McpGateway.Core.Health;
using McpGateway.Core.Proxy;
using McpGateway.Management.Services;
using McpGateway.Persistence;
using McpGateway.Telemetry;
using Microsoft.AspNetCore.Authentication;
using Microsoft.AspNetCore.Authorization;
using Microsoft.AspNetCore.Server.Kestrel.Core;
using Microsoft.EntityFrameworkCore;
using Microsoft.Extensions.Diagnostics.HealthChecks;

var builder = WebApplication.CreateBuilder(args);

// Core services (tool store, spec fetch/diff, tool generator, spec refresher)
builder.Services.AddMcpCore();

// Health services (readiness, in-flight tracker, dependency probes)
builder.Services.AddMcpHealth(builder.Configuration);

// Persistence (EF Core + PostgreSQL + repositories)
builder.Services.AddMcpPersistence(builder.Configuration);

// Management API services
builder.Services.AddMcpManagement();

// Tool call proxy (HttpRequestBuilder, ResponseWrapper, MetaToolsHandler, ToolCallHandler, Polly)
builder.Services.AddMcpProxy();

// Telemetry (OpenTelemetry traces + metrics → Dynatrace)
builder.Services.AddMcpTelemetry(builder.Configuration);

// Audit (queue emitter + disk fallback + retry worker)
builder.Services.AddMcpAudit(builder.Configuration);

// TimeProvider (used by audit + telemetry)
builder.Services.AddSingleton(TimeProvider.System);

// Auth schemes: Entra ID admin role claim + dev/test fallback
builder.Services.AddAuthentication(AdminAuthHandler.SchemeName)
    .AddScheme<JwtBearerOptions, AdminAuthHandler>(AdminAuthHandler.SchemeName, _ => { })
    .AddScheme<DevelopmentAdminOptions, DevelopmentAdminAuthHandler>(DevelopmentAdminAuthHandler.SchemeName, _ => { });

builder.Services.AddAuthorization(options =>
{
    options.AddPolicy("Admin", policy =>
        policy.AddAuthenticationSchemes(AdminAuthHandler.SchemeName, DevelopmentAdminAuthHandler.SchemeName)
              .RequireAuthenticatedUser());
});

// Built-in EF Core health check
builder.Services.AddHealthChecks()
    .AddDbContextCheck<McpGateway.Persistence.McpGatewayDbContext>(
        name: "ef_core_dbcontext",
        failureStatus: HealthStatus.Unhealthy,
        tags: new[] { "ready" });

// Kestrel hardening
builder.Services.Configure<KestrelServerOptions>(options =>
{
    options.AddServerHeader = false;
    options.Limits.MaxRequestBodySize = 10 * 1024 * 1024;
});

// Lifetime options
builder.Services.Configure<HostOptions>(options =>
{
    options.ShutdownTimeout = TimeSpan.FromSeconds(35);
    options.ServicesStartConcurrently = false;
    options.ServicesStopConcurrently = false;
});

var app = builder.Build();

app.UseAdminExceptionHandler();
app.UseAuthentication();
app.UseAuthorization();

app.MapHealthEndpoints();
app.MapAdminEndpoints();
app.MapMcp();

app.MapGet("/", () => Results.Text("MCP Gateway", "text/plain"));

await app.RunAsync();

public partial class Program { }
```

**4. Known follow-ups for Oracle review:**

- The `WebApplicationFactory<Program>` pattern requires `Program` to be a public class with a public static `Main` method. If the API project uses top-level statements (the default in .NET 6+), the compiler generates an internal `Program` class. Add the following to `src/McpGateway.Api/Program.cs` or create a `src/McpGateway.Api/PublicProgram.cs` to expose it:

  ```csharp
  public partial class Program;
  ```

- The `PUT /admin/servers/{name}/spec` endpoint may not exist with that exact path. If the management API plan defines a different path, adjust `SpecRefreshSteps.UploadSpec()` accordingly.

- The `BeforeScenario` hook in `TestRunHooks` re-creates a factory, WireMock server, and Entra ID mock per scenario. If multiple scenarios need to share state (e.g., the Entra ID mock), move it to a class fixture or to `BeforeTestRun`. The current design favors isolation over speed.

- The `TestContext` is shared across all step classes in a scenario via Reqnroll's context injection. If `ScenarioDependencies` does not compile, fall back to parameterless constructors in each step class and use a static dictionary keyed by `ScenarioContext` to share state.

- The `WhenAdminRegistersServer` step uses a hardcoded `auth_strategy: "static"`. To test the OBO strategy in `AuthStrategies.feature`, the step that updates the auth strategy must re-register or patch the server. The `UpdateServerAuthStrategy` helper in `AuthStrategiesSteps` re-POSTs to `/admin/servers`, which may create a duplicate. If duplicates are rejected, replace with a `PATCH /admin/servers/{name}` call.

- Reqnroll 2.4.0 has an `xUnit` integration package that must be referenced (it is — `Reqnroll.xUnit 2.4.0`). The integration generates xUnit `[Fact]` attributes per scenario. Each scenario becomes a test method; lifecycle hooks run per scenario.

- The BDD tests assume a clean database. The `BeforeScenario` hook applies migrations but does not truncate tables between scenarios. If previous scenarios leave data, add `await context.Database.ExecuteSqlRawAsync("TRUNCATE ai_gateway.mcp_server_defs CASCADE")` in `BeforeScenario` after migrations.

- The plan mocks the upstream API via `WireMockServer.Start()` which binds to a random port. The `base_url` passed to the admin endpoint is `_context.UpstreamApi!.Url!`. If the gateway's `HttpClient` resolves the base URL lazily and caches DNS, ensure the test does not share an `HttpClient` across scenarios.
